## 3、互联网协议

```
协议：就是一种合法的规则，前提需要双方进行规划同意


网络协议：就是我们对信息进行管理，已经规范我们的数据传输。这个规则需要我们互相通讯共同遵守

OSI（理想化模型）：ISO（国际标准组织背书（靠山的意思））把网络分7层

物理层（mac地址），数据链路层，网络层，传输层， 会话层， 表示层，应用层


```

```
从上往下进行解析
应用层： 为程序提供网络服务接口

表示层： 将上一层数据进行转换为网络数据（数据流）

会话层：负责建立，管理，终止会话（沟通）

传输层：将上面的数据进行分段处理（分包）

网络层：负责子网的数据包进行路由选择


数据链路层： 就是数据打包

物理层：将打包好的数据传输到硬件设备物理传输


```

## 4.互联网协议

```
通讯协议：网络通讯的基础

ip协议：是支持网络之间互联的数据包协议，是提供网络之间行为的一个标志

TCP/ip 协议 ：规定一种可靠的数据信息传递服务，与IP协议功能类似，可以分开，也可以合并使用的

在实际中两个协议可以进行互补的，也是互联网最常见的基础协议族（TCP/ip协议族）


tcp/ip协议又将网络分为4层

网络接口层， 网络层， 传输层， 应用层



```

```
应用： http，Https(s代表是ssl协议， 安全传输协议)，DNS(解析域名，网址解析为ip地址)，FTP（文件传输协议）

传输层：TCP，UDP
网络层：ip，icmp
网络接口层：WAN，LAN


HTTP协议：超文本（超级文本， 可以传输各种信息，包括mp3，mp4）传输协议，常用浏览器和服务器之间传输协议

```

## 5、IP地址

````
ip地址： 就是ip协议提供一种统一的地址格式，是每一个主机地址和网络地址结合

每一个ip地址包含两个识别码（id），就是网络id和主机id

查看ip地址：windows ：ipconfig
	mac，linux：ifcogfig， ip a
	

ipv4：是目前都在用ip地址，用十进制表示
ipv6：传说中要推广神秘ip地址，是以十六进制进行的


ip地址的分类：

A类：
00000000.00000000.00000000.00000000

有一个字节是网络地址和三个主机地址，网络最高位必须是0
网络地址126个，主机地址：256*2-2

范围：1.0.0.0 ~ 126.255.255.254

B类：
10000000.00000000.00000000.00000000
有两个字节是网络地址和两个主机地址，网络最高位必须是10

地址范围：128.0.0.0 ~ 191.255.255.254


C类：
11000000.00000000.00000000.00000000
有三个字节是网络地址，一个字节为主机地址，网络地址最高位必须是110
网络地址范围：192.0.0.0 ~ 223.255.255.254

D类：
多点广播： 范围224.0.0.0 ~ 239.255.255.254


E类：
这个地址是用于测试开发，范围： 240.0.0.0 ~ 255.255.255.254

255.255.255.255 广播地址


如果你想本电脑搭建一个服务，你必须要拿到公网ip 

内网 =》 网络穿透


````

## 9、网页数据请求

```
在http协议中，网页数据请求有两种格式：get请求和 post请求

get请求：发送数据的时候会显示url链接上面

优点：请求快，用于一些查询上面
缺点：不安全，数据裸露，字符串长度有限制，不同的浏览器限制长度是不一样的

post 请求： 不会显示在浏览器地址上面
而且可以发送任何数据类型，也没有数据长度限制

请求慢




```

## 10、响应码

```
响应： 1**
该响应码，就是临时识别响应码，需要请求在继续响应的响应码

100 识别服务器收到请求一部分，正在等待其他部分
101请求数据要求服务器切换协议， 服务器确认

响应 2**
200 服务器已经处理完请求数据了
201 请求成功并且完成下一次资料链接


响应： 3**
响应码表示完成请求，需要等待下一步操作

响应码 4**

就是请求失败
400 服务器无法解析语法
401 请求要求身份验证，注意出现登录业务
403 服务器拒绝请求
404 服务器找不到请求对应的网页
408 服务器等待请求超时

响应 ： 5**
服务器内部发生错误
